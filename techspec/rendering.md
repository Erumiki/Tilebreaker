# Пайплайн рендеринга

Техническая спецификация системы рендеринга Tilebreaker.

## Обзор

Весь рендеринг идёт через WebGL. DOM-элементы не используются для игровых визуалов, включая UI.

```
PNG-ассеты -> Текстурный атлас -> Спрайт-батч -> Экран
```

## Текстурный атлас

Все PNG-ассеты должны загружаться в общий атлас до начала сцены. Не биндить отдельные текстуры для каждого спрайта при отрисовке.

## Спрайт-батч

Все игровые объекты, тайлы, эффекты и UI-элементы должны рисоваться через батч квадами. Цель — минимизировать количество draw call.

## Конвенция именования ассетов

Все файлы ассетов именуются в нижнем регистре с подчёркиваниями:

```
tile_blue.png
ball_default.png
button_start.png
```

Не использовать пробелы, дефисы, CamelCase, UPPERCASE и смешанный регистр.

## UI-конфиги

UI-значения не хардкодятся в JS. Размеры, позиции, цвета, шрифты, отступы и пути к текстурам должны храниться в JSON-конфигах.
